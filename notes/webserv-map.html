WebServ — how the code flows

A non-blocking HTTP/1.1 server driven by a single poll() loop. Follow it top to bottom: a config file becomes objects, objects become listening sockets, the loop turns connections into responses.

Health check

Taking form — a real webserv, not going nuts.

The architecture is sound and recognizable: clean config→objects→listeners→loop→request separation, the correct single-poll() non-blocking design, and the right virtual-host listener model. The rough edges are in the request/response parsing (a couple of real bugs) and routing isn't wired yet — normal for this stage. Boxes with an amber/red note below flag the specific things to fix.
